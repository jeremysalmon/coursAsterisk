2. Les plans d'appels

Comme nous l'avons vu dans le "Passer un premier appel", des extensions SIP sans plan d'appel (dialplan) sont complètement inutiles.

Dans la déclaration d'un téléphone dans sip.conf la directive context = moncontexte indique à notre téléphone SIP quel diaplan il a la droit d'utiliser.

Les plans d'appels (diaplan) sont créés dans le fichier /etc/asterisk/extensions.conf. Vous pourrez trouvez une aide sur la syntaxe des diaplans dans la documentation de Asterisk

La structure d'un context est sous la forme :
[nomDuContext]
exten => nom, priorite, application(parametres)
exten => nom, priorite+1, application(parametres)

  • nom : generalement le numéro que nous composerons sur notre téléphone. Dans le TP1 nous avions créé le numéro 900
  • priorite : l'ordre d'execution de notre ligne dans le nom.
  • application : l'application et ses paramètres que nous executerons à cette ligne

Comme nous l'avons vu lors du précédemment, un nom peut être composé de multiple lignes (priorite). Ceci nous permet d'imaginer que nous utilions des briques de légo quand nous réalisons nos dialplans.

Un ensemble complet d'applications est disponible avec l'installation standard de asterisk. Pour avoir la liste des applications, il vous suffit de taper dans la console d'Asterisk core show applications

Si vous désirez l'aide sur une application particulière vous pouvez :

  • Dans la console d'Asterisk taper core show application NOM_DE_L_APPLICATION
  • Googler "nom de l'application cmd asterisk". Vous tomberez généralement dans les premiers résultats sur le site voip-info.org qui est une source d'informations précieuse.

Dans la console d'Asterisk vous pouvez voir le contenu de tout le diaplan en tapant dialplan show

Si vous voulez uniquement afficher le contenu d'un context particulier il vous suffit de taper dialplan show nomDuContext

Créer un diaplan pour un numéro 900 qui utilise les 3 applications suivante :

  • Answer() : cette application permet de décrocher l'appel
  • Noop(Ceci est un test de diaplan) : cette application permet d'afficher un message dans la console d'Asterisk
  • Hangup() : cette application raccroche un appel
N'oubliez pas que vous devez taper reload dans la console d'Asterisk pour que celui-ci prenne en considération vos changements.

Tester votre numéro 901 en le composant depuis votre softphone.

Le résultat doit être l'affichafe de "Ceci est un test de dialplan" dans la console de Asterisk.
