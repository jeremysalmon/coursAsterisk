Asterisk Database

Asterisk contient en interne une base de données qui lui permet de stocker des données même si vous redémarrez votre serveur.

Cette base est utilisée pour stocker de nombreuses données, dont par exemple les informations du REGISTRAR qui enregistre l'adresse IP liée à un compte enregistré.

Vous pouvez interagir avec la base de données de Asterisk à l'aide de la console ou à partir des plans d'appels

La base est basée sur une notion de famille, clé et valeur.

Par exemple math/jeremy/10 indique que Jeremy a eu 10 en math et math/omar/15 indique que omar a eu 15 en math.

Utilisation de la console

  • database show : affiche le contenu entier de la base de données
  • database put FAMILLE CLE VALEUR : insère une donnée dans la base de données (ex database put math jeremy 10)
  • database get FAMILLE CLE : récupère une donnée dans la base de donnée (ex database get math jeremy)
  • database del FAMILLE CLE : supprime une FAMILLE CLE de la base de donnée (ex database del math jeremy)

Utilisation avec le dialplan

Vous pouvez utilisez le plan d'appel pour manipuler la base de données de Asterisk.

  • Set(DB(FAMILLE/CLE)=VALEUR) : insère une donnée dans la base
  • Set(mavariable=${DB(FAMILLE/CLE)}) : créé une variable dans le diaplan avec une valeur récupérée dans la base de données
  • DBdel(FAMILLE/CLE) : supprime une FAMILLE CLE de la base de donnée

Comme vous l'avez vu avec la fonction Set(mavariable=${DB(FAMILLE/CLE)}), vous pouvez créer des variables dans le plan d'appel de Asterisk.
Ces variables sont ensuites accessibles grâce à la syntaxe ${NOM_VARIABLE} dans le dialplan.

De nombreuses variables se créées au moment de vos appels. Vous pouvez avoir une liste de celles ci a cette page : http://www.voip-info.org/wiki/view/Asterisk+variables

A FAIRE :
Créer un dialplan qui permet à une standardiste de dire si elle est présente ou pas.
Quand la standardiste arrive elle compose un numéro (ex : 800) pour dire à la base de données qu'elle est présente.
Quand elle part elle compose un numéro (ex : 801) pour dire à la base de donnée qu'elle part.
Si quelqu'un essaye de l'appeler quand elle n'est pas là, on doit tomber directement sur sa boite vocale.

Conseils :

  • Schématisez avec un algorithme les différentes étapes
  • Utiliser la fonction GotoIf
