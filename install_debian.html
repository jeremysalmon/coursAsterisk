Installation de Debian et des services Minimum

Le premier écran vous demande de sélectionner une langue. Idéalement, si vous parlez un minimum l'anglais, je vous conseille d'utiliser cette langue d'installation.

En effet, il vous sera ainsi plus facile de "googler" des messages d'erreur et de trouver plus facilement des réponses.

La localisation n'est pas importante pour nous.

Ici sélectionnez le Francais pour avoir un clavier AZERTY.

L'installeur charge alors tous les modules présent dans le CD de la NetInstall. Dans la grande majorité des cas, ces modules seront suffisant.

Dans le cas d'un "vrai" serveur physique, vous pouvez trouver des problèmes à partir d'ici. Par exemple dans la non prise en charge d'un contrôleur de disque dur ou d'une carte réseau.

Indiquer le nom local de votre machine.

Pour notre TP nous n'avons pas besoin de domaine. Vous pouvez laisser ce paramètre vide

Le compte root est le compte administrateur de la machine qui a tous les droits.

Indiquer un mot de passe (idéalement 1234) car c'est une machine de TP.

ATTENTION : Ne jamais mettre de mot de passe de ce type sur un serveur de production !

Confirmez votre mot de passe.

Debian vous créé un compte utilisateur par défaut.

Indiquez ici le nom complet de l'utilisateur (ex : Jeremy Salmon)

Indiquez le login de cette utilisateur (sans espace, ni caractères spéciaux)

Indiquez le mot de passe 1234

ATTENTION : Ne jamais mettre de mot de passe de ce type sur un serveur de production !

Confirmez votre mot de passe.

Le serveur configure automatiquement à l'aide du protocole NTP s'il a accès à Internet.

Le processus de partitionnement du disque dur commence

C'est ici que le disque dur sera formatté et prêt à recevoir le système d'exploitation.

Sélectionnez "Use entire disk" pour formatter le disque de 8Go

Sélectionnez le disque proposé.

Ce disque est notre fichier VDI.

Sélectionnez "All files on one partition". Ceci ne créera q'une seule partition de données.

Dans un serveur de production on sépare généralement les répertoires dans des partitions séparées.

Par exemple en mettant /var/ (toutes les données variables) dans une partition, (/bin/) les binaires ... dans une autres, etc.

Ceci permet en cas de remplissage des logs par exemple (/var/log/) ne pas remplir TOUT le disque et ainsi de faire planter la machine.

IMPORTANT : Dans un serveur de production, séparez les répertoires dans des partitions dédiées.

Sélectionnez "YES" pour appliquer et donc formatter votre disque dur

Une fois le disque formatté, Debian commence à installer ses fichiers de base sur le disque dur.

Avant cette étape, aucune écriture sur le disque dur n'avait eut lieu.

Le système de gestion de paquets APT à besoin d'un "annuaire" d'applications à récupérer sur Interner.

L'idéal est de sélectionner un serveur prêt de nous. Dans notre cas la France par exemple.

Dans chaque pays il y a plusieurs "copies" de cet annuaire. Sélectionnez un serveur.

Dans notre cas nous n'avons pas de proxy sur le réseau.

Debian télécharge l'annuaire, le décompresse et l'installe sur notre serveur.

L'étude statistique permet d'envoyer sur les serveurs de Debian des informations sur tous les paquets que vous installez, mettez à jour, supprimez, etc.

Dans notre cas nous n'enverrons pas nos données

Nous devons maintenant choisir les paquets que nous installerons pour démarrer.

Notre serveur n'aura pas d'interface graphique et nous n'utiliserons que la console.

Déselectionnez avec la barre espace les paquets inutile et ne conservez que "SSH Server" et "standard system utilities"

L'installeur téléchargera alors les logiciels et les installera.

Enfin nous devons installer le "Boot Loader" qui permettra à notre Linux de démarrer.

Sélectionnez "Yes"

Sélectionnez /dev/sda

Votre Debian va maintenant démarre et être prêt à utiliser.
